# 🚀 PROJECT PROMETHEUS · GitHub Pages 배포 가이드

> 목표: 지금 만든 HTML을 **무료로 인터넷에 올려**, 카페·노션에서 링크로 열 수 있게 만들기.
> 소요 시간: **약 5~10분**. 코딩 지식 필요 없음. (전부 웹 화면에서 클릭만)
> 준비물: **GitHub 계정**(무료) + **index.html**(이 폴더에 함께 드린 파일)

---

## 0단계 · GitHub 계정 만들기 (이미 있으면 건너뛰기)
1. https://github.com 접속 → **Sign up**.
2. 이메일·비밀번호·아이디(username) 입력. → 이 **username**이 나중에 주소에 들어갑니다. (예: `storm`)
3. 이메일 인증까지 완료.

---

## 1단계 · 저장소(Repository) 만들기
1. 로그인 후 우측 상단 **`+`** → **New repository**.
2. **Repository name**: 영문/숫자/하이픈으로. 예) `power-empire`
3. 공개 범위: 반드시 **Public** 선택.
   > ⚠️ 무료 GitHub Pages는 **Public 저장소에서만** 됩니다. (Private은 유료 플랜 필요)
4. 나머지 옵션(README 추가 등)은 체크하지 않아도 됩니다.
5. **Create repository** 클릭.

> 💡 팁: 저장소 이름을 `username.github.io`(예: `storm.github.io`)로 만들면 주소가 `https://storm.github.io/`처럼 짧아집니다(계정당 1개만 가능). 스터디용이면 그냥 프로젝트 저장소로 충분합니다.

---

## 2단계 · index.html 업로드
1. 새로 만든 저장소 첫 화면에서 **"uploading an existing file"** 링크 클릭.
   (안 보이면: **Add file ▾ → Upload files**)
2. 함께 드린 **`index.html`** 을 드래그해서 놓거나 **choose your files**로 선택.
3. 아래로 스크롤 → **Commit changes** 클릭.

> ⚠️ 파일 이름은 반드시 소문자 **`index.html`** 이고, 저장소 **최상단(루트)** 에 있어야 합니다.
> (다른 이름이면 주소 뒤에 파일명을 붙여야 열립니다. `index.html`이면 주소만으로 바로 열림)

---

## 3단계 · GitHub Pages 켜기
1. 저장소 상단 메뉴 **Settings** 클릭.
2. 왼쪽 사이드바에서 **Pages** 클릭.
3. **Build and deployment** 항목에서
   - **Source**: **Deploy from a branch** 선택
   - **Branch**: **main** 선택 / 폴더는 **/(root)**
4. **Save** 클릭.
5. **1~2분** 기다립니다.

---

## 4단계 · 접속 & 확인
- 완성 주소 형식:
  ```
  https://<username>.github.io/<저장소이름>/
  ```
  예) username `storm`, 저장소 `power-empire` →
  **`https://storm.github.io/power-empire/`**
- **Settings → Pages** 상단에 `Your site is live at ...` 로 주소가 뜹니다. 클릭해서 접속!
- 📱 모바일에서도 같은 주소로 열립니다(반응형 지원).

### 404가 뜨거나 안 열릴 때 체크리스트
- [ ] 1~2분 더 기다리기 (첫 배포는 시간이 걸립니다)
- [ ] 파일명이 정확히 `index.html`(소문자)인지
- [ ] 저장소가 **Public**인지
- [ ] Settings → Pages의 Branch가 **main / (root)** 인지
- [ ] 저장소 상단 **Actions** 탭에서 배포(초록 체크) 완료됐는지

---

## 5단계 · 내용 업데이트(수정 반영)
- **디자인·기본 데이터(파일 자체)** 를 바꾸려면:
  저장소에서 `index.html` 클릭 → 연필(**Edit**) 아이콘 → 수정 → **Commit** →
  **또는** Add file → Upload files로 새 `index.html`을 올려 덮어쓰기.
  → **1~2분 후 자동으로 사이트에 반영**됩니다.
- 앞으로 제가 기능을 더 추가하면, 새 `index.html`로 **교체 업로드만** 하면 끝입니다.

---

## 6단계 · 카페 / 노션에 연결
- **네이버 카페**: 글쓰기 화면에 위 주소를 **링크**로 붙이기.
  (카페는 앱을 안에 심을 수 없어 링크로 연결합니다.)
- **노션**: 페이지에서 `/embed` 입력 → 주소 붙여넣기 → **페이지 안에서 바로** 지도가 보입니다. (또는 링크)

---

## 7단계 · 여러 명이 같이 쓰기 ⭐ (꼭 읽어주세요)
이 앱에서 **체크·편집한 내용은 "그 사람의 브라우저에만" 저장**됩니다(localStorage).
즉, A가 정복 체크를 해도 B의 화면에는 자동으로 보이지 않습니다. (같은 앱, 각자 데이터)

하나의 **"공식본"** 을 함께 쓰는 3가지 방법:

1. **운영자 단독 관리 (추천·가장 간단)**
   운영자 1명이 앱에서 편집 → 상단 **💾 저장(JSON 내보내기)** →
   그 데이터를 `index.html`의 기본값(SEED)에 반영해 **재업로드**.
   → 모두가 접속하면 **같은 최신 상태**를 봅니다. (원하시면 JSON을 저에게 주세요. 파일에 반영해 드립니다.)

2. **JSON 공유**
   운영자가 내보낸 `.json` 파일을 카페/노션에 올려두고, 각자 상단 **📂 불러오기**로 로드.

3. **구글시트 연동(시세)**
   ETF·TOP PICK의 현재가는 구글시트 CSV로 **전원 동일하게 자동 갱신**됩니다.
   (`=GOOGLEFINANCE("KRX:종목코드","price")` → 파일·공유·웹에 게시·CSV → 그 URL을 앱에 입력)

> 🔧 더 편하게: **"GitHub의 data.json을 자동으로 읽어와 전원이 같은 데이터를 보는 동기화 버튼"** 을
> 앱에 추가할 수 있습니다. 그러면 운영자가 GitHub에서 data.json만 갱신하면 모두 최신본을 봅니다.
> 필요하시면 다음 단계로 넣어 드릴게요.

---

## 8단계 · (선택) 나만의 도메인 연결
- 도메인이 있다면: Settings → Pages → **Custom domain**에 입력 →
  도메인 업체 DNS에 A레코드 4개 추가:
  `185.199.108.153` / `185.199.109.153` / `185.199.110.153` / `185.199.111.153`
- 필수는 아닙니다. `github.io` 기본 주소로도 충분합니다.

수정은 파일 교체 업로드, 공유는 카페·노션에 링크.
